Git Hub (REMADE): Nautuls.git

Underwater light horror game where you pilot a sub and drevers ruins of a by gone age.
This was a nine week project with 3 programmers, artist 4 and 3 designers.

Controbution

I worked on 3 different aspects of the game, altho one was left finnished but unimplemented.

I made the sonar sytem that is used in the game to help the player avoid crashing into walls and other obsticals.
It was oringaly made the idea that I would get the data from the world and bascily translate all of it to a texture to avoid using a camera but do to how diffcult that was to make and how few programmers there were we went back to using a camera instead.
In the end the logic behind the sonar and all of its functnaltiy was made by me but it was optimzed by a teamate of mine knows as Kjell Hopskin, see his work here .

I also made the dialog sytem that is in the game. It has a type writer effect, plays audio and can be called from multiple different events.
It is a simple to use tool for the desginers to get the needed dialog into the game. If I had had the time I would have made the text auto translate into the voice seeing how we used a robot voice as the ship ai.

Finnaly I made the ships hazard event sytem, I was given this task late into the projects life cyckle and was told that it might not be put in before realse.
I did however finnish one of the hazards the electricity one. I used a simple 5* scripts soultion;
1. A partent script that had 2 funtion in them On and Off.
2. Childeren scripts that used the On and Off functions, overrote them, and implmented their needed custome behavior.
3. A Mangare script that on start gahtered all scripts that inharited from the partent script
4. A Call script that told the manager to tell all of the scripts it had gahtered to call their Off functions as needed
5. A Fix it script that just Called On instead of Off

Enigine: Unity

Git Hub (REMADE): TerrainGen.git

My attepemted at making a mauntain genertor using diffusion limited aggregation.
The idea came from this video.
This was a 9 week project.

Controbution

This one was a lot to work throught so I recomend Complete breakdown for this one.

I worked with the video mention in Summary to make this. The video game me the guide line to the idea but no code to speak off so figuring out how to do everything in Unity was a fun challange.
To break down exactly what I did:
1. I generate an 8x8 texture and add a white pixel into it and store their cords in a array of structs.
2. I create x amount of more white dots arounded it filling in any of the previus white dots neighborring pixles. Storing their data into the array as well
3. Then I calcualte their color value, how white they should be, by going up a neighbor tree. The feature up the tree the pixel are the whiter it should be
4. Then I scale the image up to be double the size in two diffrent ways. One sharp way, basicly just moving their cords like this (x*2, y*2). I also make a blur verrsion, just a classic blur
5. I then use the sharper image to repeat step 2 - 4.
6. When ive reached my disred size I combine blured and sharp
7. then I put it in as the height map for the terrain.
